# Тестовое задание - C# Developer
Перед выполнением задания необходимо ознакомиться с файлами проекта. 

Основные классы, с которыми придется работать:
- `DemoBattleInfo` - содержит список боевых единиц, которые заспавнятся.
- `UnitInfo` - базовый класс со статами боевой единицы.  
- `MeleeAssassinUnitInfo`, `MeleeHedgehogUnitInfo`, и тд - содержат статы боевой единицы определенного типа  
- `UnitLogic` - базовый скрипт, который описывает поведение боевой единицы.
- `MeleeAssassinUnitLogic`, `MeleeHedgehogUnitLogic`, и тд - скрипты, которые описывают поведение юнитов в бою.

## Задание 1 - Реализовать механику оглушения.  
**Оглушение** - это некоторое состояние юнита, при котором он не может ничего делать, в том числе не получает ману. Снимается после 3х ходов или после получения 100 урона за ход. Каждое последующее наложенное оглушение обнуляет счетчики.  
  
## Задание 2 - Модифицировать способность `RangeArcher`
 После того как накопилась мана, `RangeArcher` атакует одну ближайшую цель и с шансом 25% может убить её. Если цель не умерла, то она должна получить оглушение.  

### Описание поведения RangeArcher
Юнит данного типа двигается к противнику пока не достигнет дистанции для атаки. Если дистанция позволяет атаковать, то `RangeArcher` атакует ближайшего противника. Имеет способность: атакует одну ближайшую цель и с шансом 25% может убить её.  
  
## Задание 3 - Прочитать описание и сравнить с реализацией  

Необходимо прочитать описание, сравнить с реализацией скрипта `MeleeHedgehogUnitLogic`. Всё ли реализовано по описанию? Поправить ошибки.  

### Описание поведения `MeleeHedgehog`
Боевая едица ближнего боя. Имеет способность: после каждого добавления маны накапливает +10 урона и по активации этой способности весь накопленный урон разряжает в ближайшего противника, после чего начинает заново накапливать. Способность действует только в ближнем бою на расстоянии атаки. При получении за раз критического значения урона лечится на 200 ХП. Критическим значением считается более 25% максимального запаса здоровья.

  
## Задание 4  - Добавить юнита в игру по описанию.  
### Описание поведения `MeleeMonk`
Боевая едица ближнего боя. Имеет 1800 здоровья, скорость бега - 3, урон от атаки - 5, реген маны - 5, дистанция атаки - 2. При спавне вешает на себя щит с прочностью 100. Если в юнита приходит урон, то щит поглащает 50%. Регенерация маны усиливает ранее созданный щит на прочность равной этой мане. Если щит сломался, то он взрывается, нанося ближайшему противнику 250 урона и вылечивает владельца щита на 100 ХП. После этого щит уже не накапливает прочность, т.к. был разбит. Если боевая единица умерла раньше, чем щит сломался, то вылечивает ближайшего союзника на текущее значение прочности этого щита. При получении урона текущая боевая единица лечится на 10% от полученного урона в конце своего хода (округление до ближайшего большего целого числа). Лечение от получаемого урона игнорирует состояние оглушения (добавленное в первом задании).
